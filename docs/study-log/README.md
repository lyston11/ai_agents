# 学习记录说明

这个目录用于记录每天的真实学习情况，以及每天的任务和完成状态。

记录原则：

- 只写当天真实完成的学习、阅读、编码、复盘内容。
- 不提前写“计划完成”或“应该学会了”的内容。
- 如果某个知识点只是一知半解，要明确写成“已接触，但未吃透”。
- 如果当天只是看了代码、做了规划，也照实记录，不拔高。
- 每条记录尽量附上对应文件，方便后续复盘和面试准备。
- 每天的“导师任务”和“你的完成情况”分开记录，避免任务和结果混在一起。
- 完成状态以你的真实反馈为准，不默认完成。

推荐记录结构：

1. 今日时间段
2. 今日任务
3. 完成情况
4. 实际完成内容
5. 今天确认理解的点
6. 仍然模糊的点
7. 对应代码/文件
8. 明天要做什么

维护方式：

- 我每天给你任务后，我们把任务写进当天日志。
- 你学习完后，把完成情况、卡点、实际进度反馈给我。
- 我根据你的反馈更新当天日志，并调整第二天任务。
- 如果当天没有完成原计划，就记录实际完成情况和原因。
- 后面做面试复盘时，这个目录会直接变成你的学习证据链和项目成长轨迹。
