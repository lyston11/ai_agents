## 第一部分：先把 RAG 讲清楚

### 1. 先记最短定义

`RAG = Retrieval-Augmented Generation`

中文先这样理解：

`先检索外部知识，再基于检索结果生成答案`

### 2. 为什么需要 RAG

因为大模型有三个问题：

- 不一定知道你的私有知识
- 有知识也可能答得不准
- 容易幻觉

所以 RAG 的思想不是“再训练一个模型”，而是：

`回答前先查资料`

### 3. RAG 最小流程

今天你只记 5 步：

1. 文档切块
2. 给每个块做 embedding
3. 用户问题也做 embedding
4. 去向量库里检索相近内容
5. 把检索结果拼到 prompt 里让模型回答

### 4. 你今天对 embedding 只需要理解到这层

`embedding 的作用是把文本变成可用于语义检索的向量表示`

注意：

- embedding 不是生成答案
- embedding 主要服务于“召回相关内容”
