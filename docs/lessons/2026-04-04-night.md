## 第一部分：先建立认知，不急着看代码

这一部分最重要。

### 1. 什么是 LangChain

你先把它理解成：

`LangChain 是一套搭 LLM 应用的组件库`

它主要帮你做这些事：

- 调模型
- 管提示词
- 接工具
- 接向量库
- 做输出结构化

它更像“积木库”。

### 2. 什么是 LangGraph

你先把它理解成：

`LangGraph 是在 LangChain 之上，用来做有状态流程编排的框架`

它擅长的是：

- 把 Agent 系统拆成多个节点
- 规定节点之间怎么流转
- 根据状态决定下一步去哪
- 支持循环、分支、记忆、人工介入

如果你现在觉得抽象，就先记一句：

`LangChain 更像组件库，LangGraph 更像流程控制器`

### 3. 什么是 RAG

先记最朴素的定义：

`RAG = 先检索外部知识，再让模型基于检索结果生成答案`

核心原因：

- 大模型不一定知道你的私有知识
- 大模型可能会幻觉
- 你需要让答案尽量基于文档，而不是凭空猜

### 4. RAG 最小链路

你先记住 5 步：

1. 把文档切成小块
2. 把每块文本变成向量，也就是 `embedding`
3. 用户提问时，也把问题变成向量
4. 去向量库里找最相近的文本块
5. 把这些文本块作为上下文交给模型生成答案

### 5. embedding 是什么

先不要往数学深处钻。

你先把它理解成：

`embedding 是把文本映射成可以比较语义相似度的数字向量`

它的作用不是直接回答问题，而是：

- 让“语义相近”的文本在向量空间里更接近
- 让系统可以检索到“看起来不是同一句话，但意思相近”的内容

一句话版本：

`embedding 负责把文本变成可检索的语义表示`
